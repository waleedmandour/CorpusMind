## 1. ما هو CorpusMind؟

CorpusMind هو برنامج بحثي متخصص في تحليل corpora اللغوية والخطاب متعدد
الوسائط. يتيح للباحث تحميل النصوص وإجراء عمليات البحث (concordance)،
وحساب الترابطات (collocations) والمفاتيح (keyness)، وتحليل القواعد
والخطاب، والعمل مع النصوص العربية باستخدام أدوات CAMeL.

يتضمن البرنامج أيضاً جناحاً للتحليل البصري (Vision Suite) لتحليل الصور
باستخدام القواعد البصرية لـ Kress وvan Leeuwen، والتحليل متعدد
الوسائط لربط الصور بالنصوص.

يعمل المساعد الذكي (AI Assistant) بالإجابة على أسئلة الباحث عن طريق
استدعاء أدوات التحليل والاستشهاد بأدلة محددة (أرقام أسطر concordance،
إحصائيات محسوبة). كل إجابة إما "مدعومة بأدلة" (grounded) أو موضحة
بوضوح كـ "غير مدعومة" (ungrounded).

يعمل البرنامج بالكامل على جهازك. لا تغادر أي بيانات جهازك إلا إذا
اخترت صراحةً استخدام مزود سحابي للذكاء الاصطناعي.

### واجهة ثنائية اللغة (عربي/إنجليزي)

يدعم CorpusMind واجهة مستخدم ثنائية اللغة بالكامل. يمكن التبديل بين
العربية والإنجليزية بنقرة واحدة على زر التبديل في الشريط العلوي.
عند تفعيل الواجهة العربية، يتم عكس اتجاه التخطيط بالكامل من اليمين
إلى اليسار (RTL)، ويتم تطبيق الخط العربي المناسب (Noto Naskh Arabic
أو Amiri)، وتظهر جميع قوائم الشريط الجانبي والعناوين والأزرار
باللغة العربية باستخدام لغة أكاديمية رسمية مناسبة للباحثين اللغويين.

## 2. التثبيت

تحتاج إلى ثلاثة أشياء:

1. **Python 3.12 أو أحدث** من python.org
2. **Node.js 20 أو أحدث** من nodejs.org
3. **Ollama** من ollama.com (لكي يعمل المساعد الذكي محلياً)

بعد تثبيت Ollama، افتح موجه الأوامر ونفذ:

```
ollama pull llama3.2:3b
```

### إعداد المحرك (Engine)

```
cd CorpusMind/engine
python3 -m venv .venv
source .venv/bin/activate
pip install -e ".[dev]"
python -m spacy download en_core_web_sm
corpusmind-engine
```

### إعداد واجهة الويب

افتح موجه أوامر جديد:

```
cd CorpusMind/web
npm install
npm run dev
```

افتح http://localhost:5173 في متصفحك.

### دعم اللغة العربية (اختياري)

```
pip install camel-tools pyrsistent muddler cachetools emoji future regex
camel_data -i morphology-db-msa-r13
camel_data -i dialectid-model6
```

## 3. إنشاء مشروع وتحميل النصوص

1. انقر على **Projects** في الشريط الجانبي.
2. انقر على **+ New** لإنشاء مشروع جديد. أدخل اسماً واختر اللغة.
3. انقر على المشروع لتحديده.
4. انقر على **+ New** تحت Corpora لإنشاء corpus.
5. اسحب وأفلت الملفات النصية في منطقة التحميل.

الصيغ المدعومة: TXT, DOCX, PDF, HTML, XML, CSV, Markdown.

يقوم المحرك تلقائياً بكشف الترميز، وتنظيف النص، والتقسيم إلى كلمات
(tokenization)، والوسم بأجزاء الكلام (POS tagging)، والاسترجاع
(lemmatization)، والتحليل النحوي (dependency parsing) باستخدام spaCy.

## 4. البحث في Concordance (KWIC)

1. حدد corpus.
2. انقر على **Concordance** في الشريط الجانبي.
3. اكتب استعلام البحث.
4. اختر المستوى: كلمة (word)، أصل (lemma)، أو وسم جزء الكلام (POS).
5. حدد نافذة السياق (عدد الكلمات المعروضة يميناً ويساراً).
6. انقر على **Search**.

يحصل كل سطر نتيجة على معرف ثابت (مثال: `doc:0:3`) يمكن للمساعد الذكي
الاستشهاد به. استخدم `*` كحرف بدل.

انقر على **Export Excel** لتحميل النتائج.

## 5. التكرار، الترابط، المفاتيح، والتشتت

**التكرار (Frequency)**: اختر أداة تحليل من الشريط الجانبي. اختر كلمة
أو أصلاً أو وسم POS. يعرض الجدول التكرار، التكرار لكل مليون كلمة،
والنسبة المئوية. يظهر STTR (نسبة النوع إلى الرمز المعيارية) في الأعلى.

**الترابط (Collocation)**: اكتب كلمة محورية (node)، حدد حجم النافذة
(افتراضي: 5 كلمات لكل جانب)، وانقر على Compute. تظهر جميع المقاييس
السبعة: MI, T-score, log-likelihood, Dice, LogDice, chi-square,
Delta P (كلا الاتجاهين).

**المفاتيح (Keyness)**: حدد corpus الهدف، ثم اختر corpus مرجعي. تظهر
النتائج مع مقاييس الأهمية (log-likelihood, chi-square) ومقاييس حجم
الأثر (Log Ratio, %DIFF, Simple Maths, Odds Ratio) جنباً إلى جنب.
انقر على **Methods PDF** لإنشاء فقرة منهجية تلقائياً لبحثك.

**التشتت (Dispersion)**: اكتب مصطلحاً. تظهر النتائج مع D لـ Juilland
(0 إلى 1، كلما اقترب من 1 كان التوزيع أكثر تساوياً) و DP لـ Gries
(0 إلى n-1/n، كلما اقترب من 0 كان التوزيع أكثر تساوياً).

## 6. أدوات التحليل المتقدمة

**N-grams**: اختر n (من 2 إلى 10). حدد الحد الأدنى للتكرار والمدى
(عدد الوثائق المختلفة التي يجب أن يظهر فيها). يتبع هذا معيار
التكرار والمدى لـ Biber et al. (1999).

**تحليل POS**: عرض توزيع أجزاء الكلام أو n-grams لأجزاء الكلام
للتحليل الأسلوبي.

**القواعد (Grammar)**: كشف المبني للمجهول، الأفعال الناقصة، النفي،
الجمل الوصفية، الجمل الاسمية المعقدة، والزمن من التحليل النحوي.

**الاعتمادية (Dependency)**: الاستعلام عن علاقات محددة (nsubj, obj,
iobj, obl, إلخ) للعثور على أزواج الفعل والفاعل/المفعول الشائعة.

**الخطاب (Discourse)**: تطبيق تصنيف Hyland (2005) لـ metadiscourse:
الانتقالات، علامات الإطار، التحييد (hedges)، المعززات (boosters)،
علامات الموقف، الإشارات الذاتية، علامات التفاعل، والمزيد.

**المفردات (Vocabulary)**: تصنيف corpus إلى نطاقات تكرار (K1, K2-K9,
AWL, Off-list). تحديد الكلمات النادرة والكلمات الأكاديمية.

**التحليل العاطفي (Sentiment)**: درجة المشاعر لكل جملة (-1 إلى +1)
مع مخطط زمني.

**الاستعارة (Metaphor)**: توليد مرشحين للاستعارة (أفعال مع فواعل
مجردة). هذه مرشحون فقط. يقوم LLM بفرزها، ويجب على الباحث التحقق
منها قبل اعتبار أي مرشح استعارة مؤكدة.

## 7. التحليل العربي

انقر على **Arabic Tools** في الشريط الجانبي. الأدوات المتاحة:

- **التحليل الصرفي**: تحليل كامل مع الجذر (al-jizr)، الوزن (al-wazn)،
  الأصل (lemma)، POS، الجذع (stem)، transliteration لـ Buckwalter،
  العدد، الجنس، وعلم الجمع المكسر. يستخدم CAMeL Tools (calima-msa-r13).
- **الجذور**: استخراج الجذور الثلاثية. مثلاً، جميع الكلمات التي تشترك
  في الجذر ك.ت.ب: كتاب، مكتبة، كاتب، يكتب.
- **تحديد اللهجة**: تحديد MSA، المصرية، الخليجية، أو الشامية باستخدام
  CAMeL DIDModel6 (6 لهجات حضرية).
- **Buckwalter**: تحويل الحروف العربية إلى لاتينية ASCII.
- **إزالة التشكيل**: إزالة الحركات (harakat).
- **التطبيع**: توحيد variations الألف، التاء المربوطة، والألف المقصورة.
- **السجل**: كشف الكلاسيكي، MSA، أو العامية.
- **الترجمة**: البحث عن equivalents الترجمة من العربية إلى الإنجليزية.

## 8. جناح التحليل البصري (Vision Suite)

انقر على **Vision Suite** في الشريط الجانبي.

1. أنشئ مجموعة صور داخل corpus.
2. اسحب وأفلت الصور (JPG, PNG, TIFF, WebP).
3. أضف تعليقات اختيارية.
4. حدد صورة واستخدم التبويبات:

**Analyse**: تحليل الألوان (الألوان السائدة، توازن الدافئ/البارد،
السطوع، التباين، التشبع، ملاحظات الرمزية)، تحليل التكوين (قيمة
المعلومات: Given/New، Ideal/Real، centre/margin، salience، rule of
thirds، التوازن البصري، التأطير)، واستخراج نص OCR.

**Visual Grammar**: تحليل الصورة وفق metafunctions الثلاث لـ Kress و
van Leeuwen (2006): التمثيلي، التفاعلي، والتكويني. كل ادعاء يُصاغ
كفرضية: "Under a Kress and van Leeuwen reading, X may indicate Y."

**Align**: محاذاة متعددة الوسائط بين الصورة والنص. اكتب النص المرافق
وانقر على Align. يستخرج المحرك مناطق الصورة وامتدادات النص، ثم
يطابقها بدرجات ثقة. يتم كشف العلاقات عبر الوسائط (تعزيز، تكامل، صمت).

**Discourse**: 8 تحليلات: التحليل الاجتماعي السيميائي، CDA (4 أطر:
Fairclough، van Dijk، Wodak، Machin وMayr)، الإقناع (Aristotle +
Toulmin)، التأطير (Entman)، السرد (Labov)، الاستعارة البصرية،
العاطفة المركبة، والتحليل الثقافي.

## 9. المساعد الذكي

انقر على **AI Assistant** في الشريط الجانبي. المساعد هو وكيل يستخدم
الأدوات (tool-using agent)، وليس chatbot. عندما تطرح سؤالاً، يختار
ويستدعي أداة التحليل المناسبة، ثم يكتب إجابته بناءً على مخرجات الأداة.

إذا استدعى المساعد أداة، تُعلام الإجابة بـ **grounded** (شارة خضراء).
إذا لم يتم استدعاء أداة، تُعلام بـ **ungrounded** (شارة برتقالية). لا
تعرض الواجهة أبداً إجابة غير مدعومة كحقيقة.

أسئلة مثال:
- "ما هي أكثر 10 كلمات تكراراً في هذا corpus؟"
- "ابحث عن جميع تكرارات 'research' واعرض سياقاتها."
- "ما هي أقوى collocates لـ 'dog' في حدود 5 كلمات؟"
- "قارن هذا corpus مع corpus المرجعي. ما هي المفاتيح الرئيسية؟"
- "ما هي hedges التي يستخدمها هذا المؤلف؟"

## 10. التصدير والتعاون والخصوصية

**التصدير**: Excel لـ concordance والتكرار والترابطات والمفاتيح. PDF
لفقرة المنهجية المسودة تلقائياً (تستشهد بالأدوات والإصدارات والصيغ
الدقيقة).

**عمليات البحث المحفوظة والإشارات المرجعية**: حفظ الاستعلامات مع
المعاملات لإعادة الاستخدام. وضع إشارة مرجعية على أسطر concordance
أو إحصائيات محددة مع ملاحظات.

**مشاركة المشروع**: تمييز مشروع كمشترك (عام أو خاص) مع رمز مشاركة.
يتم تسجيل أحداث المزامنة في سجل تدقيق.

**التشفير عند الراحة**: تشفير اختياري AES-256-GCM لملفات الصور على
القرص. التفعيل عبر متغير البيئة `CORPUSMIND_ENCRYPTION_KEY`.

**إمكانية الوصول**: هدف WCAG 2.1 AA. مؤشرات تركيز مرئية، رابط تخطي
إلى المحتوى، وضع التباين العالي، دعم الحركة المخفضة، أهداف لمس بحد
أدنى 44px، وعكس كامل RTL للعربية.

## 11. تطبيق الويب التقدمي (PWA) مقابل تطبيق سطح المكتب

يصدر كوربَس مايند في صورتين تشتركان في واجهة المستخدم نفسها، وفي
صيغة ملف المشروع نفسها، وفي محرك التحليل نفسه. تختلف الصورتان في
كيفية تسليم المحرك، وموقع البيانات، والميزات المتاحة. توثّق هذه
القسم الفروق بدقة ليتمكن الباحثون من اختيار الصورة التي تناسب
سير عملهم وقيود حوكمة البيانات المؤسسية.

### 11.1 ما هو تطبيق الويب التقدمي

تطبيق الويب التقدمي (PWA) هو نسخة المتصفح من كوربَس مايند المستضافة
على https://corpus-mind-web.vercel.app/. يمكن تثبيته على Chrome وEdge
وSafari وFirefox بالضغط على أيقونة التثبيت في شريط العنوان. بعد
التثبيت، يعمل في نافذته الخاصة، ويعمل دون اتصال بعد الزيارة الأولى،
ويظهر في قائمة تطبيقات نظام التشغيل.

في إعداد PWA، يعمل محرك التحليل إما كخدمة بعيدة (عندما يكون لدى
الباحث وصول إلى محرك كوربَس مايند مستضاف ذاتياً على خادم مختبر) أو
كعملية يبدأها الباحث محلياً بأمر `corpusmind-engine` ثم يتصل بها من
المتصفح. لا يُشغّل المتصفح نفسه بايثون؛ بل يتواصل مع المحرك عبر
HTTP على `127.0.0.1:8765` أو على عنوان بعيد يُعدّه الباحث.

### 11.2 ما هو تطبيق سطح المكتب

تطبيق سطح المكتب هو ملف تنفيذي Tauri 2 لـ Windows وmacOS (Apple
Silicon وIntel) وLinux. يجمع واجهة الويب وعملية الإشراف بلغة Rust
و(عند البناء بمواصفات السايدكار) المحرك المُجمَّع بـ PyInstaller في
حزمة تثبيت واحدة. عند الإطلاق، يُنشئ المشرف بلغة Rust المحرك كعملية
تابعة، ويستعلم نقطة نهاية الصحة حتى يصبح جاهزاً، ويُعيد توجيه مخرجاته
القياسية والأخطاء إلى ملفات سجل، ويُنهيه بنظافة عند خروج التطبيق.

### 11.3 جدول مقارنة الميزات

| المساعد الذكي (نموذج محلي عبر Ollama) | يتطلب من الباحث تشغيل Ollama محلياً وإعداد عنوان المحرك | يمكن للمشرف تشغيل Ollama تلقائياً إذا كان على مسار النظام |
| المساعد الذكي (نموذج بعيد عبر OpenAI / Anthropic) | مدعوم، بالاشتراك الاختياري | مدعوم، بالاشتراك الاختياري |
| تسليم المحرك | يبدأ الباحث المحرك يدوياً، أو يتصل بمحرك خادم مختبر | يُنشئ المشرف المحرك ويشرف عليه تلقائياً |
| التشفير عند الراحة (AES-256-GCM) | مدعوم، مع مفتاح التشفير عبر متغير البيئة | مدعوم، مع المفتاح عبر مربع حوار إعدادات سطح المكتب أو البيئة |

| التشغيل دون اتصال | بعد الزيارة الأولى، تُحمَّل قشرة PWA من الذاكرة المؤقتة؛ لكن المحرك لا يزال بحاجة للتشغيل محلياً للتحليل | دون اتصال بالكامل بعد التثبيت؛ لا حاجة لاتصال شبكي لأي عملية تحليلية |

| الاتساق عبر المنصات | متطابق على كل نظام تشغيل لأن المتصفح هو بيئة التشغيل | واجهة متطابقة، لكن المشرف والملف التنفيذي للسايدكار خاصان بالمنصة (بناء واحد لكل نظام تشغيل) |
| التثبيت | نقرة واحدة (أيقونة التثبيت في المتصفح) | تنزيل وتشغيل مثبّت المنصة (.dmg، .exe، .msi، .AppImage، .deb) |
| آلية التحديث | شفافة؛ يُحدَّث PWA عند الزيارة التالية عندما يُسلِّم الخادم نسخة جديدة | يدوي؛ يُنزِّل الباحث المثبّت الجديد أو يبني من المصدر |
| العزل الز Sandbox | عزل المتصفح يُقيّد وصول نظام الملفات والشبكة بما تسمح به سياسة أمان المحتوى | تُفرض قشرة Tauri نفس السياسة، لكن المشرف بلغة Rust لديه صلاحيات تحكم كاملة في العمليات |
| النشر المؤسسي | سهل: وجّه PWA إلى عنوان محرك خادم مختبر وشارك العنوان مع الفريق | يُثبّت كل باحث تطبيق سطح المكتب؛ خادم المختبر اختياري |
| موقع إقامة البيانات | إذا كان المحرك على خادم مختبر، تعيش بيانات المجموعة على ذلك الخادم؛ إذا كان محلياً، تعيش على جهاز الباحث | تعيش بيانات المجموعة دائماً على جهاز الباحث؛ لا تغادر البيانات الجهاز ما لم يُعدَّ مزوّد نماذج بعيد |

### 11.4 أي صورة تختار

اختر **PWA** إذا أردت تجربة كوربَس مايند دون تثبيت أي شيء، أو إذا
كانت مؤسستك تستضيف بالفعل محركاً مشتركاً على خادم مختبر، أو إذا كنت
تعمل عبر أجهزة كثيرة وتريد أن تتبعك حالة مشروعك. PWA هو أيضاً الخيار
الوحيد على ChromeOS وعلى أجهزة الشركات المقفولة حيث لا يمكنك تثبيت
تطبيقات أصلية.

اختر **تطبيق سطح المكتب** إذا كنت تعمل مع مجموعات لغوية حساسة أو غير
منشورة وتحتاج إلى إعداد محلي صارم افتراضياً، أو إذا أردت أن يبدأ
المحرك ويتوقف تلقائياً مع التطبيق، أو إذا أردت إدارة سجلات المحرك
نيابة عنك، أو إذا احتجت التجربة الكاملة دون اتصال دون وجود متصفح في
الدائرة. تطبيق سطح المكتب هو أيضاً الصورة الموصى بها للنشر في
الفصول الدراسية حيث يُثبّت المُحاضِر التطبيق مسبقاً على أجهزة المختبر
ولا يحتاج الطلاب إلى إدارة دورة حياة المحرك.

## 12. مقارنة كوربَس مايند بأدوات تحليل اللغة الأخرى

كوربَس مايند ليس بديلاً عن كل أداة تحليل لغوي موجودة. كل أداة في
هذا القسم صُمِّمت لجمهور مختلف ولحقبة مختلفة من عصر الحوسبة. تقارن
هذه القسم كوربَس مايند بأربع أدوات تحليل لغوي مستخدمة على نطاق واسع:
AntConc وSketch Engine و#LancsBox وVoyant Tools. تستند المقارنة إلى
الوثائق الرسمية لكل أداة حتى عام 2025، وكل ادعاء مصدر إلى موقع الأداة
نفسه أو إلى منشور محكّم عنها.

### 12.1 AntConc

AntConc هو مجموعة أدوات تحليل لغوي مجانية طورها لورانس أنتوني في
جامعة واسيدا وصدر أول مرة عام 2002. هو تطبيق سطح مكتب مكتوب بلغة
Perl ومُجمَّع إلى ملف تنفيذي أصلي لـ Windows وmacOS وLinux. يُوزَّع
مجاناً من https://www.laurenceanthony.net/software/antconc/.

يوفّر AntConc أداة كشف سياقي (KWIC)، وأداة عناقيد وn-grams، وأداة
متضايق، وأداة قائمة كلمات، وأداة كلمات مفتاحية. يدعم البحث
بالتعبيرات النمطية، ومعالجة الوسوم المُعرَّفة من المستخدم، ومجموعة من
خيارات فرز المخرجات. لا يتضمن مؤشّر تصريف صرفي، أو واسم فئة نحوية،
أو محللاً تبعياً؛ يُتوقع من الباحث معالجة المجموعة اللغوية مسبقاً
بأداة خارجية إذا لزم البحث بالتصريف أو بالفئة النحوية. لا يتضمن
مساعداً ذكياً اصطناعياً، أو خط معالجة بصري، أو تحليلاً متعدد الوسائط.
دعم العربية محدود لأن AntConc لا يضم محللاً صرفياً عربياً؛ يعالج
الباحثون العاملون بالعربية المجموعة عادةً بأداة منفصلة مثل Farasa أو
CAMeL Tools ثم يستوردون النص المُوسوم إلى AntConc.

يختلف كوربَس مايند عن AntConc في أربعة أوجه. أولاً، يضم كوربَس مايند
spaCy وCAMeL Tools، فالتشفير والتأصيل الصرفي والوسم النحوي والتحليل
التبعي والصرف العربي متاحة دون أي خطوة معالجة مسبقة. ثانياً، ينفّذ
كوربَس مايند جميع مقاييس التضايق السبعة الموثقة في أدبيات المنهجية
(MI، T-score، الاحتمالية اللوغاريتمية، Dice، LogDice، مربع كاي،
Delta P) وكلا مقياسي التمييز (نسبة الاحتمالية اللوغاريتمية وZeta
لباروز) في واجهة واحدة، بينما تُصدر أداة المتضايق في AntConc إحصائية
واحدة يختارها المستخدم. ثالثاً، يضم كوربَس مايند مجموعة الرؤية لتحليل
الخطاب متعدد الوسائط ومساعداً ذكياً اصطناعياً بعقد إلزامي بالاستشهاد،
لا يوفر AntConc أيهما. رابعاً، يُصدر كوربَس مايند سجل إثبات YAML لكل
عملية تحليلية، بينما لا يسجّل AntConc إعداد خط المعالجة أو إصدار
البرنامج مع المُخرج. يبقى AntConc خياراً ممتازاً لتدريس اللسانيات
اللغوية على المستوى التمهيدي، وللبحث السياقي السريع على المجموعات
المعالجة مسبقاً، وللباحثين الذين يفضّلون أداة سطح مكتب أحادية الغرض
دون تبعيات شبكية.

### 12.2 Sketch Engine

Sketch Engine منصة تجارية لإدارة المجموعات اللغوية وتحليلها طورتها
Lexical Computing Limited وصدرت أول مرة عام 2003. هي خدمة قائمة على
الويب متاحة على https://www.sketchengine.eu/. تتطلب اشتراكاً مدفوعاً؛
تبدأ الحسابات الأكاديمية الشخصية من حوالي 7.09 يورو شهرياً، وترخيصات
المؤسسات تُسعَّر لكل مستخدم. تتوفر طبقة مجانية باسم SKELL لمتعلمي
اللغة لكنها لا تتضمن مجموعة الأدوات التحليلية الكاملة.

الميزة المميزة لـ Sketch Engine هي Word Sketch: ملف تعريف تضايق
نحوي من صفحة واحدة، يُولَّد تلقائياً لكلمة، بُني بتحليل المجموعة
بقواعد رسم خاصة باللغة وتجميع المتضايقين حسب العلاقة النحوية (فاعل
لـ، مفعول لـ، مُعدِّل لـ، إلخ). تقدّم أيضاً قاموساً توزيعياً، وكشف
سياقي، وقائمة كلمات، وكلمات مفتاحية، وn-grams، واستخراج مصطلحات.
تستضيف Sketch Engine مجموعة كبيرة من المجموعات الجاهزة (المجموعة
الوطنية البريطانية BNC، ومجموعة الويب الإنجليزية 2021، وenTenTen،
وغيرها الكثير) وتسمح للباحثين برفع مجموعاتهم حتى حد حجم يحدّده مستوى
الاشتراك. تدعم أكثر من 100 لغة، مع واسمات وقواعد رسم خاصة باللغة
لمعظمها.

يختلف كوربَس مايند عن Sketch Engine في ثلاثة أوجه. أولاً، كوربَس مايند
محلي أولاً ومجاني (AGPL-3.0-only)؛ لا تغادر مجموعة الباحث جهازه أبداً
ما لم يُعدّ صراحةً مزوّد نماذج بعيد. Sketch Engine خدمة سحابية؛ تُرفع
المجموعة إلى خوادم Sketch Engine للمعالجة. هذا يجعل Sketch Engine غير
مناسبة للمجموعات غير المنشورة أو السرية ما لم يكن لدى الباحث ترخيص
مؤسسي مع اتفاقية معالجة بيانات. ثانياً، لا ينفّذ كوربَس مايند حالياً
ملف تعريف التضايق النحوي Word Sketch؛ هذه ميزة على خارطة الطريق لإصدار
لاحق. يبقى Word Sketch في Sketch Engine التنفيذ المرجعي لتلخيص
التضايق النحوي، وينبغي للباحثين الذين يحتاجونه استخدام Sketch Engine.
ثالثاً، يضم كوربَس مايند مجموعة الرؤية لتحليل الخطاب متعدد الوسائط
ومساعداً ذكياً اصطناعياً بعقد إلزامي بالاستشهاد؛ لا تقدّم Sketch Engine
أيهما. Sketch Engine الخيار الأفضل للباحثين الذين يحتاجون مجموعات
مرجعية كبيرة جاهزة، ولعلماء المعجم الذين يعتمدون على Word Sketch،
وللفرق التي تفضّل خدمة سحابية مُدارة على تثبيت محلي.

### 12.3 #LancsBox

#LancsBox صندوق أدوات تحليل لغوي مجاني طور في جامعة لانكستر بواسطة
فاكلاف بريزينا وزملائه وصدر أول مرة عام 2015. النسخة الحالية،
#LancsBox X، متاحة على https://lancsbox.lancs.ac.uk/. هي تطبيق سطح
مكتب لـ Windows وmacOS وLinux يُوزَّع مجاناً للاستخدام الأكاديمي غير
التجاري.

يوفّر #LancsBox كشفاً سياقياً، وتحليل تضايق، وقوائم تردد، وتحليل كلمات
مفتاحية، ومخططات تشتت، وتصور "Graph Coll" المميز الذي يُظهر علاقات
التضايق كرسم شبكي بالكلمة المحورية في المركز. يتضمن دعماً مدمجاً
لمجموعات BNC وBNC64 المرجعية ويسمح للباحثين برفع مجموعاتهم. يدعم
لغات متعددة للتشفير لكنه لا يضم واسمات فئة نحوية خاصة باللغة أو
محللات صرفية بعد الإنجليزية.

يختلف كوربَس مايند عن #LancsBox في ثلاثة أوجه. أولاً، ينفّذ كوربَس
مايند مجموعة التضايق الكاملة ذات المقاييس السبعة ومجموعة التشتت ذات
المقاييس الأربعة (D لجويلاند، DP لجريس، التردد المخفّض المتوسط، وقت
الانتظار المتوسط) مع سجلات إثبات، بينما يركّز #LancsBox على مجموعة
أصغر من المقاييس مع تصور أغنى. ثانياً، يضم كوربَس مايند مجموعة الرؤية
والمساعد الذكي؛ لا يفعل #LancsBox. ثالثاً، يضم كوربَس مايند CAMeL Tools
للصرف العربي وتحديد اللهجة والتعرف على الكيانات، بينما يعامل #LancsBox
العربية كمجموعة نصوص عادية دون تحليل صرفي. #LancsBox الخيار الأفضل
للباحثين الذين يقدّرون تصور Graph Coll الشبكي، للاستخدام في الفصول
الدراسية حيث الحس البصري المباشر للأداة يدعم التدريس، وللباحثين
الذين يعملون بشكل رئيسي مع مجموعات BNC وBNC64 المرجعية.

### 12.4 Voyant Tools

Voyant Tools بيئة تحليل نصي مجانية قائمة على الويب طورها ستيفان
سنكلير وجيفري روكويل وصدرت أول مرة عام 2012. متاحة على
https://voyant-tools.org/. هي مفتوحة المصدر ويمكن استضافتها ذاتياً،
لكن معظم الباحثين يستخدمون النسخة المستضافة العامة.

يوفّر Voyant Tools واجهة متعددة الألواح فيها قارئ، وسحابة كلمات،
ورسم اتجاهات، وأداة سياقات، وأداة متضايق، وأداة ارتباطات، ومخطط
انتشار، وتصور حالم الصورة. صُمِّمت للعمل في العلوم الإنسانية الرقمية
على المجموعات الصغيرة والمتوسطة وتُعطي أولوية للاستكشاف البصري على
الصرامة الإحصائية. لا تضم واسم فئة نحوية، أو مؤشّر تصريف، أو محللاً
تبعياً؛ تعمل على تردد الرمز الخام. لا تنفّذ مقاييس التضايق القياسية
(MI، T-score، الاحتمالية اللوغاريتمية، إلخ) ولا تُصدر سجل إثبات. لا
تدعم الصرف العربي.

يختلف كوربَس مايند عن Voyant Tools في أربعة أوجه. أولاً، ينفّذ كوربَس
مايند المجموعة الإحصائية الكاملة (سبعة مقاييس تضايق، أربعة مقاييس
تشتت، مقياسا تمييز) بتعريفات رسمية واستشهادات بالأدبيات الأولية،
بينما يركّز Voyant Tools على الاستكشاف البصري. ثانياً، يضم كوربَس مايند
spaCy وCAMeL Tools للوسم اللغوي، بينما يعمل Voyant Tools على الرموز
الخام. ثالثاً، يضم كوربَس مايند مجموعة الرؤية والمساعد الذكي؛ لا يفعل
Voyant Tools. رابعاً، يُصدر كوربَس مايند سجل إثبات YAML لكل عملية؛
لا يفعل Voyant Tools. Voyant Tools الخيار الأفضل للباحثين في العلوم
الإنسانية الرقمية الذين يريدون نظرة عامة بصرية سريعة لنص، ولتدريس
تحليل النص التمهيدي، وللباحثين الذين لا يحتاجون وسماً لغوياً أو صرامة
إحصائية.

### 12.6 متى تستخدم أي أداة

استخدم **كوربَس مايند** عندما تحتاج تحليلاً محلياً أولاً، قابلاً لإعادة
الإنتاج، مؤسَّساً على الأطر النظرية، لمجموعات إنجليزية أو عربية، أو
عندما تحتاج تحليل خطاب متعدد الوسائط للصور إلى جانب النص، أو عندما
تحتاج مساعداً ذكياً اصطناعياً يؤسّس ادعاءاته بأدلة المجموعة اللغوية.

استخدم **AntConc** عندما تحتاج كشفاً سياقياً سريعاً على مجموعة
معالجة مسبقاً في بيئة فصل دراسي أو ورشة عمل، أو عندما لا تحتاج
تأصيلاً صرفياً أو وسم فئة نحوية، أو عندما تكون على جهاز لا يمكنك
تثبيت بايثون عليه.

استخدم **Sketch Engine** عندما تحتاج مجموعات مرجعية كبيرة جاهزة، أو
عندما تحتاج ملف تعريف التضايق النحوي Word Sketch، أو عندما يفضّل
فريقك خدمة سحابية مُدارة وتسمح سياسة حوكمة بياناتك برفع المجموعة إلى
خادم طرف ثالث.

استخدم **#LancsBox** عندما تقدّر تصور Graph Coll الشبكي، أو عندما
تدرّس اللسانيات اللغوية وتريد أداة بصرية بديهية، أو عندما تعمل
بشكل رئيسي مع BNC أو BNC64.

استخدم **Voyant Tools** عندما تريد نظرة عامة بصرية سريعة لنص في
سياق علوم إنسانية رقمية، أو عندما لا تحتاج وسماً لغوياً أو صرامة
إحصائية، أو عندما تعمل في بيئة متصفح فقط.

هذه الأدوات متكاملة، لا متنافية. سيستخدم كثير من الباحثين أكثر من
واحدة: مثلاً، Voyant Tools لنظرة عامة بصرية أولية، وكوربَس مايند
لتحليل إحصائي ومتعدد الوسائط صارم، وSketch Engine لـ Word Sketch
عند الحاجة إلى ملف تعريف التضايق النحوي.
