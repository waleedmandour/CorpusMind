1 What is CorpusMind?

CorpusMind is a research-software product that lets a linguist go from raw texts and images to publication-ready quantitative and qualitative analysis without writing a line of code, without sending unpublished data to a third-party server unless they explicitly choose to, and without losing the methodological transparency that peer review demands.

It ships as two integrated suites: CorpusMind Text (concordancing, frequency, collocation, keyness, n-grams, dispersion, vocabulary, POS, grammar, dependency, discourse, metaphor, sentiment, Arabic-first-class support) and CorpusMind Vision (image ingestion, OCR, colour/composition analysis, Visual Grammar per Kress and van Leeuwen, multimodal image-text alignment, CDA, persuasion, framing, narrative, emotion, cultural analysis).

Both suites are reachable from a Progressive Web App (installable, offline-capable) and from native desktop apps for Windows, Linux, and macOS (built with Tauri 2). Local LLM inference via Ollama and/or LM Studio lets the AI Assistant run entirely on the researcher's own machine.

2 Feature Highlights

Phase 1

Corpus Analysis Workbench

Concordance (KWIC with stable line IDs), frequency (with STTR), collocation (all 7 measures), keyness (significance + effect size always together), dispersion (Juilland's D + Gries' DP).

Phase 2

Advanced Text Analysis

N-grams with frequency-and-range criterion, POS distribution, grammar pattern detectors, dependency queries, Hyland's metadiscourse, vocabulary profiling, sentiment, MIPVU metaphor candidates.

Phase 3

Arabic First-Class Support

CAMeL Tools morphology, root extraction (al-jizr), pattern identification (al-wazn), Buckwalter transliteration, dialect ID (6-city model), broken plural detection, bilingual alignment, parallel concordance.

Phase 4

Vision Suite MVP

Image ingestion, OCR, colour analysis, composition analysis, Visual Grammar (Kress and van Leeuwen), multimodal image-text alignment (the flagship feature).

Phase 5

Multimodal Discourse

Social semiotic, CDA (4 frameworks), persuasion (Aristotle + Toulmin), framing (Entman), narrative (Labov), visual metaphor, emotion, cultural analysis. Facial analysis opt-in (off by default).

Phase 6

Collaboration and Polish

Saved searches, bookmarks, favorites, project sharing, at-rest encryption, WCAG 2.1 AA accessibility, self-hosted lab deployment with Nginx TLS.

3 Grounded AI Architecture

The AI Assistant is a tool-using agent, not a chatbot. Every empirical claim must come from a tool call. If no tool was invoked, the answer is visibly flagged as ungrounded. This is the load-bearing implementation of the grounded-AI principle.

25 registered tools across all phases: search_concordance, get_frequency, compute_collocations, compute_keyness, get_dispersion, get_ngrams, get_pos_analysis, grammar_query, dependency_query, discourse_analysis, vocab_profile, sentiment, metaphor_candidates, arabic_morphology, arabic_dialect_id, arabic_roots, arabic_register, arabic_transliterate, visual_grammar, social_semiotic, cda, persuasion, framing, and ping.

Conversations persist in SQLite with full audit trail: every turn, every tool call, every evidence item. The UI renders every claim with clickable citations back to concordance lines, image regions, or computed statistics.

4 Privacy and Ethics

  • ✓Local-first by default. Corpus text, images, and AI queries never leave the user's machine unless they explicitly opt in.
  • ✓Cloud hard-disable switch. Self-hosted deployments can block all cloud routing.
  • ✓At-rest encryption. AES-256-GCM, opt-in. Key never stored on disk.
  • ✓Facial analysis opt-in. Off by default. Never performs identity recognition.
  • ✓No telemetry. No analytics, no error reporting, no phone-home.
  • ✓Framework-lensed hypotheses. Every interpretive claim is phrased as a hypothesis, never as a bare assertion.

5 Getting Started

Three commands to a running system:

cd engine && python3 -m venv .venv && source .venv/bin/activate && pip install -e ".[dev]" && corpusmind-engine

cd web && npm install && npm run dev

ollama serve && ollama pull llama3.2:3b

Then open http://localhost:5173 in your browser. Create a project, drag-drop text files, and start analyzing.
